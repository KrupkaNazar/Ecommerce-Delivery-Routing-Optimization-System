# Система оптимізації процесів доставки в електронній комерції

Інформаційно-аналітична система для оптимізації маршрутів доставки на основі математичних моделей VRP, CVRP, VRPTW та прогнозування ETA.

## Про проект

**Кваліфікаційна робота** на здобуття ступеня магістра  
Спеціальність: 122 «Комп’ютерні науки»  
Освітньо-професійна програма: «Управління інформацією та аналітика даних»  
Національний університет харчових технологій (НУХТ), Київ, 2025

**Тема роботи:** Розроблення аналітичної моделі для оптимізації процесів доставки в електронній комерції

## Мета проекту

Розробити аналітичну модель та інформаційно-аналітичну систему, яка дозволяє оптимізувати маршрути доставки з урахуванням часових вікон, обмежень місткості транспортних засобів, географічних особливостей та динамічного прогнозування часу прибуття (ETA).

## Використані технології

- **Python** — основна мова розробки
- **Google OR-Tools** — розв’язання задач VRP, CVRP, VRPTW
- **Random Forest Regressor** — прогнозування ETA
- **OSRM + Nominatim** — маршрутизація та геокодування
- **Folium (Leaflet)** — візуалізація маршрутів на карті
- **SQLite** — локальна база даних
- **CustomTkinter** — графічний інтерфейс користувача
- **scikit-learn, pandas, numpy** — обробка даних та машинне навчання

## Основні можливості системи

- Оптимізація маршрутів за моделями VRP, CVRP та VRPTW
- Прогнозування часу прибуття (ETA) за допомогою моделі машинного навчання
- Геокодування адрес та побудова матриць відстаней
- Кластеризація точок доставки (K-Means)
- Візуалізація оптимальних маршрутів на інтерактивній карті
- Облік обмежень за місткістю транспортних засобів та часовими вікнами
- Генерація аналітичних звітів і графіків ефективності

## Структура дашборду / системи

Система складається з таких основних модулів:
1. Геокодування та попередня обробка даних
2. Кластеризація адрес доставки
3. Оптимізація маршрутів (VRP / CVRP / VRPTW)
4. Прогнозування ETA
5. Візуалізація результатів на карті
6. Аналітичні звіти та графіки

## Структура репозиторію

- `reports/` — фінальний файл дипломної роботи (.doc)
- `src/` — вихідний код системи (Python)
- `data/` — тестові та реальні набори даних
- `screenshots/` — скріншоти інтерфейсу та візуалізацій маршрутів
- `docs/` — додаткові документи (ER-діаграма, структура системи тощо)

## Джерела даних

- Реальні та синтетичні логістичні дані (замовлення, адреси, часові вікна)
- OpenStreetMap (Nominatim + OSRM)
